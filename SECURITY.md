# Политика безопасности

## Сообщение об уязвимостях

Если вы обнаружили уязвимость в XKeen Control Panel, пожалуйста, сообщите об этом ответственно:

- **НЕ создавайте** публичный GitHub issue
- Используйте [GitHub Security Advisories](https://github.com/shisui1511/xkeen-control-panel/security/advisories/new)
- Включите: описание проблемы, шаги воспроизведения и потенциальное воздействие

| Этап | Целевой срок |

## Архитектура безопасности

XKeen Control Panel реализует многоуровневую модель безопасности:

### Аутентификация и авторизация

- **bcrypt** для хеширования паролей
- **HMAC** для сессионных cookie
- **CSRF-токены** для защиты от подделки запросов
- **Rate limiting** по IP (5 попыток входа, блокировка на 5 минут)

### Защита файловой системы

- Path validation: все пути проверяются через `PathValidator` на соответствие `AllowedRoots`
- Запрещён path traversal (`../`, абсолютные пути вне разрешённых корней)
- CRUD операции с конфигами только в `/opt/etc/xray`, `/opt/etc/xkeen`, `/opt/etc/mihomo`, `/opt/var/log`

### Сетевая безопасность

- Встроенный HTTP-сервер (без внешних зависимостей)
- Security headers (X-Content-Type-Options, X-Frame-Options и др.)
- CORS не требуется (backend и frontend на одном origin)
- Cookie: HttpOnly, SameSite=Strict, опционально Secure

### Логирование

- Санитизация пользовательского ввода перед записью в логи
- Защита от log injection

## Обязательные меры

- Установите пароль администратора при первом запуске
- При наличии HTTPS включите `secure_cookie: true` в `config.json`
- Ограничьте доступ к порту 8089 файрволом роутера
- Обновляйтесь через встроенный механизм self-update

## Зависимости

- CI/CD включает CodeQL семантический анализ безопасности (Go + JavaScript/TypeScript)
- Автоматические обновления через `scripts/setup.sh`
